## 서론 (훅)

"월 요금제 3일 만에 주간 한도 초과했어요. 분명 작업 방식은 바뀐 게 없는데."

이 문장을 Hacker News에서 처음 봤을 때, 덧글만 16개가 달렸다. 그리고 전부 같은 경험이었다.

Claude Code Max 플랜 사용자들이 $200/월을 내고도, 왜 토큰이 사라지는지 모른 채 한도 초과를 맞이하고 있다.

---

## 본문

### 실제로 무슨 일이 벌어지고 있나?

Hacker News에 "What are you moving on to now that Claude Code is so rate limited?" 스레드가 올라왔다 (2026년 4월 4일).

직접 인용:
- "오전 2시간 만에 주간 한도의 20% 소진"
- "1개 작은 리포에서 하루 만에 주간 할당량 전부 소비"
- "Opus로 PRD → 코드 → 테스트 워크플로우 → 엔터프라이즈 팀 플랜에서 1시간 만에 한도"

이건 특이한 케이스가 아니다. 이게 **에이전틱 코딩의 실제 비용 구조**다.

### 왜 이런 일이 생기나: 숨겨진 비용 배수

Claude Code를 "그냥 쓰면" 괜찮아 보인다. 하지만 에이전틱 워크플로우에서는 비용 배수 요인들이 조용히 작동한다:

**1. 1M 컨텍스트 창 (기본 활성화)**
HN 유저 MeetingsBrowser의 설명:
> "Turn off the 1M context that got enabled by default. Long sessions eat through the tokens much faster. Your sessions were probably getting auto-compacted much earlier before."

기본 설정이 1M 컨텍스트 창으로 바뀌었다. 이전보다 세션당 토큰 사용량이 **5-10배 증가**할 수 있다.

**2. 에이전트 루프의 복리 효과**
멀티-스텝 에이전트 태스크는 각 툴 호출이 독립적으로 컨텍스트 창을 쌓는다. 10단계 태스크 = 10배의 토큰 부하.

**3. 프롬프트 캐시 바이패스**
일부 워크플로우(특히 외부 에이전트 통합)는 Anthropic의 프롬프트 캐싱을 우회한다. 결과: API-등가 컴퓨트 비용이 $200/월 Max 플랜에서 하루 $1,000-5,000 수준으로 폭발.

### "그러면 사용량 페이지 보면 안 되나요?"

된다. 하지만 문제는 **사용량 대시보드가 이해하기 어렵고, 자주 틀린다**는 것.

HN 유저 tstrimple:
> "API로는 월 $800 사용하는데 플랜 한도는 0% 사용 중이라고 나온다. 명확하게 설명해달라는 요청을 했지만 기다리라는 말만 들었다."

Claude의 `/usage` 명령어는 플랜 소비량과 실제 API 토큰 사용량을 다르게 계산한다. 이 차이를 이해하는 개발자는 거의 없다.

### 해결책: 내가 만들고 싶었던 도구

이 문제를 해결하려고 **ClaudeTrack**을 만들고 있다.

핵심 기능:
- **실시간 세션 비용 추적**: 지금 이 세션에서 얼마 썼는지
- **예산 알림**: "월 $100의 80% 도달 시 알림"
- **세션 ROI**: 투입 비용 대비 완성된 태스크 추적
- **최적화 제안**: "이 워크플로우 패턴이 비용을 X% 증가시킵니다"

아직 초기 단계지만, 베타 신청은 여기서: **claudetrack.qucord.com**

## 마무리

에이전틱 AI 코딩은 게임을 바꿨다.

하지만 $200/월을 내고, 내 토큰이 어디 갔는지 모른다면? 그건 GPU 렌털 서버 비용 보는 것보다 더 답답한 블랙박스다.

가시성이 없으면, 최적화도 없다.

ClaudeTrack은 그 가시성을 만드는 프로젝트다.
